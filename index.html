The Block-0 Loop

Every build follows the same discipline — no exceptions.

→ Request arrives at SwarmRouter-4B
↳ classifies: domain · complexity · risk_level · proposal_required
→ proposal gate (pharma + safety: always · high-risk: required)
→ Worker Model executes the task
→ SwarmJudge-27B scores output: accuracy · completeness · structure · relevance · sft_quality
→ verdict: PASS (score ≥ 20, accuracy ≥ 4) → seal · ship · done
or FAIL → issues named → fixes prescribed → targeted retrain

Build Discipline

Block-0 is the foundation. The judge is the gate. No model ships without a sealed verdict.

Step 1

Block-0

Train fresh on sealed, SHA256-verified data. No recycled weights. Clean start every build.

Step 2

Evaluate

SwarmJudge-27B runs acceptance tests. 5 criteria × 1-5. PASS threshold: score ≥ 20, accuracy ≥ 4.

Step 3 — PASS

Seal & Ship

Block-0 IS v1. SHA256 sealed. Recorded in R2 + Supabase. Immutable build record.

Step 3 — FAIL

Diagnose & Fix

Issues named. Fixes prescribed. Targeted retrain only — no blind re-runs. That becomes v1.
