Verification Flow

How Zero-Trust Delivery Works

1
Pairs pass 6 quality gates
JSON valid, output length, numeric verify, concept present, dedup, degenerate check. Code gates — no subjective scoring.
2
SHA-256 fingerprint per pair
Every question + answer is normalized and hashed. Unique fingerprint for every training pair in the batch.
3
Merkle tree built from all fingerprints
20,000 pair hashes → binary tree → single 64-char root hash. Any pair can be verified against this root.
4
Merkle root published to Hedera HCS
One immutable message per batch. Timestamped by Hedera consensus. Cost: $0.0001. Permanent.
5
Customer receives pairs + guarantee.json
The guarantee includes Merkle root, gate pass rates, Hedera transaction ID, and a HashScan verification link.
6
Verify any pair independently
Pick any pair. Hash it. Check inclusion proof against the on-chain Merkle root. Math proves integrity. Zero trust required.
